## 새 프로젝트 생성

### STEP 1: 새 프로젝트 버튼 클릭
1. 좌측 사이드바에서 **"새 프로젝트"** 클릭
2. 또는 대시보드 우측 상단 **"+ 새 프로젝트"** 버튼 클릭

### STEP 2: 프로젝트 정보 입력

#### 2-1. 프로젝트 제목
```
예시:
- "클라우드 마이그레이션 성공 사례"
- "AI 도입으로 업무 효율 3배 향상"
- "제조업 스마트팩토리 구축 가이드"
```

**팁**: 유튜브/인스타 제목으로 바로 사용할 수 있도록 작성하세요.

#### 2-2. 원문 텍스트 입력
- **최소 길이**: 300자 이상
- **최대 길이**: 5,000자 이하
- **권장**: 1,000~2,000자 (쇼츠 최적 길이)

**입력 가능한 텍스트 유형:**
- 블로그 포스트
- 사례 연구 보고서
- 제안서 요약
- 프레스 릴리스
- 백서 일부

#### 2-3. 콘텐츠 유형 선택

| 유형 | 설명 | 예시 |
|------|------|------|
| 💡 **문제 제기형** | 업계 문제점 지적 | "왜 아직도 수작업으로 하시나요?" |
| ✅ **해결 제시형** | 솔루션 제안 | "3단계로 해결하는 데이터 보안" |
| 🏗️ **구축 사례형** | 프로젝트 성공 사례 | "LG전자 스마트팩토리 구축기" |
| ⚖️ **비교형** | A vs B 비교 분석 | "온프레미스 vs 클라우드 비교" |
| 🎯 **영업 전환형** | 상담/데모 유도 | "무료 컨설팅 신청하기" |
| 👔 **대표 브랜딩형** | 회사/대표 소개 | "바이칼시스템즈 소개" |

**선택 기준**:
- 원문의 **주요 목적**에 맞는 유형 선택
- 콘텐츠 유형에 따라 **AI 프롬프트**가 자동 최적화됩니다

### STEP 3: 프로젝트 생성
- **"프로젝트 생성"** 버튼 클릭
- 자동으로 프로젝트 상세 페이지로 이동
- AI 파이프라인이 자동 시작됩니다

## 파이프라인 실행

프로젝트 생성 후 AI 파이프라인이 자동으로 순차 실행됩니다.

### 파이프라인 단계

#### 1️⃣ 요약 (Summarization)
- **소요 시간**: 10~20초
- **처리 내용**: 원문을 쇼츠용 핵심 메시지로 압축
- **결과**: 200~300자 요약문

**진행 표시**:
```
[■■■■□□] 요약 생성 중... (Step 1/5)
```

#### 2️⃣ 기획 (Planning)
- **소요 시간**: 15~25초
- **처리 내용**: 쇼츠 구성 계획 수립 (훅, 전개, CTA)
- **결과**: 3단계 구성 플랜

#### 3️⃣ 대본 생성 (Script)
- **소요 시간**: 20~30초
- **처리 내용**: 30초~60초 분량 대본 작성
- **결과**: 음성 낭독용 스크립트

#### 4️⃣ 씬 분해 (Scene Breakdown)
- **소요 시간**: 15~25초
- **처리 내용**: 대본을 5~10개 씬으로 분할
- **결과**: 씬별 텍스트 + 지속시간

#### 5️⃣ TTS 생성 (Text-to-Speech)
- **소요 시간**: 씬당 3~5초
- **처리 내용**: OpenAI TTS로 음성 파일 생성
- **결과**: 씬별 MP3 파일 + 총 영상 길이

**전체 소요 시간**: 약 2~4분

### 실시간 상태 확인
- 각 단계별 **초록색 체크 아이콘** 표시
- 진행 중인 단계는 **회전 아이콘** 표시
- 오류 발생 시 **빨간색 에러 메시지** 표시

## 대본 및 씬 편집

### 대본 편집

#### 편집 방법
1. **"대본"** 탭 클릭
2. 대본 우측의 **연필 아이콘** 클릭
3. 텍스트 수정
4. **"저장"** 버튼 클릭

**편집 시 주의사항:**
- 음성 낭독에 적합하도록 **자연스러운 구어체** 사용
- 1문장은 **20자 이내**로 짧게 (호흡 고려)
- 전문 용어는 **한글 발음** 병기 (예: "API(에이피아이)")

### 씬 편집

#### 씬 구조
```
┌──────────────────────────────┐
│ 씬 #1 - 훅 (3초)             │
│ "이런 문제 겪고 계신가요?"   │
│ [TTS: scene_1.mp3]           │
└──────────────────────────────┘
```

#### 편집 가능 항목
- **씬 텍스트**: 음성 내용 수정
- **씬 길이**: 자동 계산 (TTS 기준), 수동 조정 불가

#### TTS 재생성
1. 씬 텍스트 수정 후 저장
2. **"TTS 재생성"** 버튼 클릭
3. 해당 씬만 새로 음성 생성 (10초 소요)

**팁**: 여러 씬을 한번에 수정한 후 일괄 재생성하세요.

## 렌더링 설정 및 실행

### 렌더링 전 체크리스트
- ✅ 대본 최종 검토 완료
- ✅ 모든 씬 TTS 생성 완료
- ✅ 총 영상 길이 확인 (권장: 30~60초)

### STEP 1: 렌더링 설정 열기
1. 프로젝트 상세 하단 **"렌더링 설정 및 시작"** 버튼 클릭
2. 렌더링 설정 다이얼로그 표시

### STEP 2: 렌더링 옵션 설정

#### 해상도 선택
| 해상도 | 비율 | 플랫폼 |
|--------|------|--------|
| **1080×1920** | 9:16 | YouTube Shorts, Instagram Reels |
| **1080×1350** | 4:5 | Instagram Feed |
| **1080×1080** | 1:1 | Instagram Post |

**권장**: 9:16 (YouTube + Instagram 모두 호환)

#### 프레임레이트 (FPS)
- **24 FPS**: 영화 느낌 (부드럽지 않음)
- **30 FPS**: 표준 (권장) ⭐
- **60 FPS**: 고품질 (파일 크기 2배)

#### 색상 설정

**배경색 (Background Color)**
```
기본값: #0A1628 (다크 블루)
권장: 어두운 색상 (가독성 향상)
```

**강조색 (Accent Color)**
```
기본값: #00D4FF (바이칼 시안)
용도: 브랜드 로고, 강조 요소
```

**자막 색상 (Caption Color)**
```
기본값: #FFFFFF (흰색)
권장: 배경과 대비 높은 색상
```

#### 자막 설정

**폰트 크기**
- 범위: 24px ~ 120px
- 기본값: 52px
- 권장: 48~60px (모바일 가독성)

**자막 위치**
- **하단**: 일반적 (권장) ⭐
- **중앙**: 강조형
- **상단**: 로고 삽입 시

### STEP 3: 렌더링 시작
1. 설정 완료 후 **"적용하고 렌더링 시작"** 버튼 클릭
2. 렌더링 큐에 추가
3. 진행 상태 실시간 표시

### 렌더링 진행 상태

#### 큐 대기 (Pending)
```
🕐 큐 대기 중... (현재 대기: 2개)
```
- 최대 2개 동시 렌더링
- 3번째부터는 대기열에 추가

#### 렌더링 중 (Rendering)
```
🎬 렌더링 중... (1/3)
```
- **소요 시간**: 씬 개수 × 5~10초
- 예시: 8개 씬 = 약 60초

#### 완료 (Completed)
```
✅ 렌더링 완료! 결과 보기
```

#### 실패 (Failed)
```
❌ 렌더링 실패: FFmpeg 오류
```
- 관리자에게 문의하세요

## 패키지 다운로드

### 패키지 구성
렌더링 완료 후 다운로드되는 ZIP 파일 구조:

```
project_title_20260315.zip
├── video.mp4              # 최종 영상
├── thumbnail.png          # 썸네일 이미지
├── metadata.json          # 메타데이터
│   ├── title
│   ├── description
│   ├── hashtags
│   └── duration
└── README.txt             # 업로드 가이드
```

### 다운로드 방법
1. **"패키지 다운로드"** 버튼 클릭
2. ZIP 파일 자동 다운로드
3. 압축 해제 후 사용

### 메타데이터 활용

#### YouTube Shorts 업로드
```json
{
  "title": "클라우드 마이그레이션 3단계 가이드",
  "description": "안전한 클라우드 전환을 위한 필수 체크리스트\n\n#클라우드 #마이그레이션 #IT컨설팅",
  "hashtags": ["클라우드", "마이그레이션", "바이칼시스템즈"]
}
```

#### Instagram Reels 업로드
- 자동 연동 사용 (다음 섹션 참고)
- 또는 수동 업로드: metadata.json의 description 복사

## 소셜 미디어 연동

### Instagram 연동

#### STEP 1: 설정 페이지 접속
1. 좌측 사이드바 **"설정"** 클릭
2. **"소셜 미디어 연동"** 섹션 이동

#### STEP 2: Instagram 계정 연동
1. **"Instagram 연동하기"** 버튼 클릭
2. 팝업 창에서 Instagram 로그인
3. 권한 승인:
   - ✅ instagram_basic
   - ✅ instagram_content_publish
   - ✅ pages_read_engagement
4. 콜백 처리 대기 (5~10초)
5. **"연동됨"** 상태 확인

**주의사항:**
- Instagram **비즈니스 계정** 필요
- 개인 계정은 연동 불가

#### STEP 3: Reels 자동 업로드
1. 프로젝트 상세 페이지로 이동
2. **"Instagram에 업로드"** 버튼 클릭 (계획 중)
3. 캡션 입력 (또는 자동 생성 캡션 사용)
4. 업로드 시작

**업로드 프로세스:**
```
1. 영상 URL → Instagram 서버 전송
2. 미디어 컨테이너 생성 (5~10초)
3. 게시물 발행 (즉시)
4. ✅ 완료: Instagram 앱에서 확인
```

## FAQ

### Q1. 영상 길이는 어떻게 조절하나요?
**A**: 대본 작성 시 텍스트 양으로 조절됩니다.
- 짧은 영상 (30초): 500자 이내 원문
- 표준 영상 (45초): 1,000자 원문
- 긴 영상 (60초): 2,000자 원문

### Q2. 원문이 너무 길면 어떻게 하나요?
**A**: 다음 방법 중 선택:
1. 원문을 직접 요약해서 입력
2. 여러 프로젝트로 분할 (시리즈 형태)
3. 핵심 섹션만 복사해서 사용

### Q3. 대본을 완전히 새로 작성할 수 있나요?
**A**: 가능합니다.
1. 대본 편집 모드 진입
2. 전체 텍스트 삭제 후 새로 작성
3. TTS 재생성 클릭

### Q4. 렌더링이 실패했어요. 어떻게 하나요?
**A**: 다음 순서로 확인:
1. 모든 씬의 TTS 생성 완료 확인
2. "다시 렌더링" 버튼 클릭
3. 2회 이상 실패 시 관리자 문의

### Q5. 여러 해상도로 동시에 렌더링할 수 있나요?
**A**: 현재는 불가능합니다.
- 해상도별로 별도 렌더링 필요
- Q2 2026 업데이트에서 일괄 렌더링 기능 추가 예정

### Q6. 음성을 다른 목소리로 변경할 수 있나요?
**A**: 현재는 고정 (OpenAI TTS 'nova' 음성)
- Q3 2026: 여러 음성 옵션 추가 예정
- Q4 2026: AI 음성 클로닝 기능 추가 예정

### Q7. 영상에 BGM을 넣을 수 있나요?
**A**: 현재는 TTS 음성만 포함
- BGM 추가 기능은 로드맵에 있습니다
- 임시 해결책: 다운로드 후 편집기에서 추가

### Q8. 브랜드 로고를 변경하고 싶어요.
**A**: 관리자에게 요청하세요.
- 로고는 서버 설정에서 일괄 변경
- 향후 프로젝트별 로고 설정 기능 추가 예정

### Q9. 프로젝트를 삭제할 수 있나요?
**A**: 현재는 불가능합니다.
- 프로젝트 상태를 "완료"로 변경 후 필터링
- Q2 2026: 삭제 기능 추가 예정

### Q10. 다른 팀원과 협업할 수 있나요?
**A**: 현재는 단일 사용자 모드
- Q3 2026: 팀 협업 기능 (코멘트, 승인 플로우) 추가 예정

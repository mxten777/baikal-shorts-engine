## 1단계: PC 부팅 및 VS Code 실행 (30초)

### 작업 순서
1. Windows 부팅
2. VS Code 실행
3. 프로젝트 폴더 열기
   - 경로: `C:\baikal_platform\baikal_shorts_engine`
   - 또는 최근 프로젝트에서 선택

## 2단계: 백엔드 서버 시작 (1분)

### PowerShell 터미널 1 열기

```powershell
cd C:\baikal_platform\baikal_shorts_engine\backend
.\venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

### ✅ 정상 실행 확인

터미널에서 다음 메시지가 표시되어야 합니다:

```
✅ 환경 변수 검증 완료
INFO:     Will watch for changes in these directories: ['C:\\baikal_platform\\baikal_shorts_engine\\backend']
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

### ❌ 오류 발생 시

#### 오류 1: 환경 변수 설정 오류

```
🚨 환경 변수 설정 오류
다음 필수 환경 변수가 설정되지 않았습니다:
  ❌ SUPABASE_URL
  ❌ OPENAI_API_KEY
```

**해결 방법**:
```powershell
notepad backend\.env
```
- `.env.example` 파일을 참고하여 누락된 변수 입력
- 저장 후 서버 재시작

#### 오류 2: 포트 8000 이미 사용 중

```powershell
# 기존 프로세스 종료
Get-Process -Name python | Where-Object {$_.Path -like "*baikal*"} | Stop-Process -Force

# 서버 재시작
.\venv\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

## 5단계: 시연 시나리오 준비 (2분)

### 시나리오 A: 새 프로젝트 생성 → 렌더링 (추천)

#### 5-1. 프로젝트 생성
1. `http://localhost:5173` 접속
2. **"새 프로젝트"** 버튼 클릭
3. 프로젝트 정보 입력:
   - **제목**: "바이칼시스템즈 IT 컨설팅 소개"
   - **원문**: 미리 준비한 샘플 텍스트 붙여넣기 (아래 참조)
   - **콘텐츠 타입**: SOLUTION
4. **"생성"** 클릭

#### 5-2. 파이프라인 자동 실행
- 프로젝트 생성 후 자동으로 상세 페이지로 이동
- 파이프라인 단계가 자동으로 실행됨:
  - ✅ 핵심 요약
  - ✅ 기획 생성
  - ✅ 대본 생성
  - ✅ 씬 분해
  - ✅ 음성 생성

#### 5-3. 생성 결과 확인
- **대본 탭**: Hook, Body, CTA 확인
- **씬 구성 탭**: 씬별 자막과 시간 확인

#### 5-4. 렌더링 시작
1. **"렌더링 시작"** 버튼 클릭
2. 렌더 설정 확인:
   - 해상도, FPS, 색상 등 기본값 사용 또는 조정
   - **썸네일 스타일 선택** (그라디언트/테크/미니멀/대담)
3. **"렌더링 시작"** 최종 클릭
4. 렌더 결과 페이지로 자동 이동
   - 진행률 표시 (0% → 100%)
   - 예상 소요 시간: 60-90초

#### 5-5. 결과 확인
- 렌더링 완료 후 비디오 플레이어 표시
- **재생** 버튼으로 생성된 쇼츠 확인
- **썸네일 이미지 미리보기 및 다운로드**
- YouTube/Instagram 캡션 확인

### 시나리오 B: 기존 프로젝트 시연

대시보드에 이전에 생성한 프로젝트가 있다면:
1. 프로젝트 카드 클릭
2. 생성된 대본/씬 설명
3. 렌더 결과 페이지 이동하여 영상 재생

## 🚨 트러블슈팅

### 문제 1: 백엔드 서버가 시작되지 않음

#### 원인 1: 환경 변수 누락
```
🚨 환경 변수 설정 오류
```

**해결**:
```powershell
notepad backend\.env
```
- `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`, `OPENAI_API_KEY` 확인
- 값이 `your-`로 시작하면 실제 값으로 변경

#### 원인 2: 포트 충돌
```
ERROR: Address already in use
```

**해결**:
```powershell
Get-Process -Name python | Where-Object {$_.Path -like "*baikal*"} | Stop-Process -Force
```

### 문제 2: 프론트엔드 화면이 로딩되지 않음

#### 원인: 백엔드 서버 미실행

**확인**:
```powershell
Invoke-WebRequest http://localhost:8000/health
```

**해결**: 2단계로 돌아가서 백엔드 서버 재시작

### 문제 3: 파이프라인이 실행되지 않음

#### 원인 1: OpenAI API 키 오류

**확인**: 백엔드 터미널에 다음과 같은 에러 표시
```
ERROR: OpenAI API error: Incorrect API key provided
```

**해결**: `.env` 파일의 `OPENAI_API_KEY` 확인 및 수정

#### 원인 2: 인터넷 연결 끊김

**확인**: Wi-Fi 아이콘 또는 네트워크 설정
**해결**: 안정적인 네트워크로 재연결

### 문제 4: 렌더링이 매우 느림 (2분 이상)

#### 원인: Pexels 이미지 다운로드 지연

**대처**: 고객에게 다음과 같이 안내
> "현재 씬별로 실제 배경 이미지를 Pexels에서 다운로드하고 있어 
> 시간이 조금 소요됩니다. 실제 프로덕션에서는 캐싱을 통해 
> 더 빠르게 처리할 수 있습니다."

**임시 해결**: `.env`에서 `PEXELS_API_KEY` 제거 (단색 배경 사용)

### 문제 5: 생성된 영상 재생 안 됨

#### 원인: 브라우저 코덱 미지원

**해결**: Chrome 또는 Edge 최신 버전 사용
